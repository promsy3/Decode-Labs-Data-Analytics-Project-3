# Project 3: SQL Data Analysis

## 📌 Project Overview
This repository contains the database queries executed for Project 3 of the Decode Labs Data Analytics Internship. The goal of this phase is to move analytical pipelines into a relational database environment (MySQL) to extract targeted business insights using formal querying syntax.

## 🛠️ Tech Stack & Skills
- **Database Engine:** MySQL Server
- **Interface Tool:** MySQL Workbench
- **SQL Skills:** DQL (Data Query Language), filtering, sorting, multi-field aggregations, and categorical grouping.

## 🗄️ File Manifest
- `project3_queries.sql`: Structured SQL script containing optimized queries fulfilling all project evaluation steps.
- `Dataset for Data Analytics .xlsx`: Source transactional data imported into the active schema environment.

## 🔍 SQL Implementation Breakdown
1. **Targeted Data Extraction:** Wrote focused `SELECT` queries filtered by strict volume parameters (`WHERE Quantity > 3`) and arranged dynamically (`ORDER BY TotalPrice DESC`) to highlight high-value sales transactions.
2. **Global Aggregations:** Leveraged mathematical function tokens (`COUNT`, `SUM`, `AVG`) to instantly extract top-level performance indicators including global transaction volumes, structural inventory totals, and macro basket values.
3. **Categorical Performance Evaluation:** Utilized explicit `GROUP BY` logic combined with chronological sorting parameters to break down transaction volume distributions and gross revenue totals across unique product lines.
